# 漏洞名称：XSS 跨站脚本漏洞

测试时间：2026.02
测试环境：DVWA  
测试人员：jingzi

---

## 一、漏洞等级

中危

---

## 二、漏洞描述

应用程序未对用户输入内容进行 HTML 实体编码，
攻击者可构造恶意 JavaScript 代码，
在其他用户浏览页面时执行。

---

## 三、漏洞复现步骤

1. 进入 DVWA XSS 模块
2. 在输入框中输入以下 payload：

```
<script>alert(1)</script>
```

3. 提交请求
4. 页面弹出 alert 弹窗

## 五、漏洞原理分析

应用未对用户输入内容进行过滤或转义。

浏览器在解析 HTML 时，将 `<script>` 标签解析并执行其中的 JavaScript 代码。

攻击流程：

1. 攻击者提交恶意脚本
2. 脚本被存储或直接返回
3. 受害者访问页面
4. 浏览器执行恶意代码

---

## 六、风险影响

- 用户 Cookie 被窃取
- 会话劫持
- 页面内容篡改
- 钓鱼攻击

---

## 七、修复建议

1. 对输出内容进行 HTML 实体编码
2. 使用安全模板引擎
3. 设置 HttpOnly Cookie
4. 启用 CSP（Content Security Policy）

---

## 八、复测标准

修复后输入：

```
<script>alert(1)</script>
```

页面应显示为普通文本：

```
&lt;script&gt;alert(1)&lt;/script&gt;
```

不得执行脚本。

---

## 九、总结

本漏洞属于典型输出未编码导致的 XSS 漏洞，
修复方式明确，应统一采用编码策略。
