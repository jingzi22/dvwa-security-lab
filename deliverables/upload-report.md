# 漏洞名称：文件上传漏洞

测试时间：2026.02
测试环境：DVWA  
测试人员：jingzi  

---

## 一、漏洞等级

高危

---

## 二、漏洞描述

应用程序未对上传文件类型进行严格校验，
攻击者可上传恶意脚本文件，
导致服务器执行恶意代码。

---

## 三、漏洞复现步骤

1. 进入 DVWA File Upload 模块
2. 构造简单 WebShell 文件：

```php
<?php phpinfo(); ?>
```

3. 将文件命名为：

```
shell.php
```

4. 上传成功
5. 访问上传路径触发执行

## 五、漏洞原理分析

服务器仅检测文件扩展名或未检测文件内容。

攻击者可通过：

- 修改文件后缀
- 修改 Content-Type
- 双写扩展名
- 解析漏洞

绕过限制。

---

## 六、风险影响

- 远程代码执行
- 服务器被控制
- 网站被挂马
- 数据被窃取

---

## 七、修复建议

1. 严格校验文件类型（白名单机制）
2. 校验 MIME 类型与文件内容
3. 上传目录禁止执行权限
4. 随机重命名文件
5. 文件存储到独立服务器

---

## 八、复测标准

修复后上传：

```
shell.php
```

应提示：

- 文件类型非法
或
- 无法执行脚本

---

## 九、总结

文件上传漏洞属于高风险漏洞，
应优先进行严格白名单校验与执行权限控制。
